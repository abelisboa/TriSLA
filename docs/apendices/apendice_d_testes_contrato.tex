\chapter{Testes de Contrato (Reprodutíveis)}
\label{ap:testes-contrato}

Os testes de contrato asseguram a conformidade entre produtores e consumidores de API.  
Os exemplos a seguir demonstram o uso reprodutível de cada tipo de interface da arquitetura.

\section*{REST}
\begin{lstlisting}[language=bash, caption={Teste REST (I-06) via curl.}]
curl -sS -X POST https://agent.core.local/v1/agent/core/reconfigure \
  -H "Authorization: Bearer $TOKEN" -H "Content-Type: application/json" \
  --data '{"policyId":"latency-guard","params":{"p99":25},"deadlineMs":1500}'
\end{lstlisting}

\section*{gRPC}
\begin{lstlisting}[language=bash, caption={Teste gRPC (I-01) via grpcurl.}]
grpcurl -d '{"intentText":"URLLC 1ms for NSI-001"}' \
  -H "Authorization: Bearer $TOKEN" \
  -insecure semcsmf.svc.local:443 trisla.v1.SemCsmfService/EvaluateIntent
\end{lstlisting}

\section*{Kafka}
\begin{lstlisting}[language=bash, caption={Publicação/consumo de evento (I-03/I-05) via kcat.}]
# Produzir
kcat -b kafka:9092 -t risk.report -k nsi-001 -P risk.json
# Consumir
kcat -b kafka:9092 -t risk.report -C -o -1 -c 1 -J
\end{lstlisting}
